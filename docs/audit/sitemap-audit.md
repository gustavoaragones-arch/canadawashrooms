# Sitemap & URL Inventory Audit — CanadaWashrooms.ca

**Audit date:** 2026-06-21  
**Production origin:** `https://canadawashrooms.ca` (apex; `www` → apex 308 redirect)  
**Local build sitemap:** `dist/sitemap.xml` (generated via `npm run sitemap` after `npm run build`)  
**Audit tooling:** `scripts/audit-seo-gsc.ts` → `docs/audit/seo-audit-data.json`

---

## Executive summary

The application defines **230 indexable URL patterns** in code. Production serves a sitemap with **81 URLs**. The local sitemap generator produces **181 URLs**. Neither matches the full expected inventory.

**Most critical finding:** Direct HTTP requests to almost all non-home routes return **404** on production because the Vercel deployment lacks SPA fallback rewrites. `_redirects` (`/* /index.html 200`) is a Cloudflare/Netlify convention and is **not applied on Vercel**. Googlebot receives 404 for sitemap URLs unless this is fixed.

## Part 2 — URL validity (production HTTP check)

Checked 2026-06-21 against `https://www.canadawashrooms.ca` (redirects to apex).

| URL | Status | Issue |
|-----|--------|-------|
| `/` | 200 | OK (after www → apex redirect) |
| `/providers/` | 404 | SPA route not rewritten to `index.html` |
| `/alberta/` | 404 | SPA route not rewritten |
| `/city/calgary/` | 404 | SPA route not rewritten |
| `/construction-jobsites/` | 404 | SPA route not rewritten |
| `/construction-portable-washrooms/calgary/` | 404 | SPA route not rewritten |
| `/provider/rhino-calgary-ab/` | 404 | SPA route not rewritten |
| `/privacy/` | 404 | SPA route not rewritten |
| `/about/` | 404 | SPA route not rewritten |
| `/coverage/` | 404 | SPA route not rewritten |

**Root cause:** Vercel serves static files only. Client-side React Router paths require a `vercel.json` rewrite (or equivalent) to `/index.html`. Current `public/_redirects` is not consumed by Vercel.

**Redirect behaviour:**

- `https://www.canadawashrooms.ca/*` → `https://canadawashrooms.ca/*` (308) — correct apex consolidation
- No redirect loops observed
- `/sitemap.xml` returns **200** on apex

**Soft 404 risk:** Vercel `NOT_FOUND` plain-text 404 responses are hard 404s, not soft 404 HTML shells — but they block indexing entirely.

## Part 3 — Canonical audit

| Page type | Self-canonical | Absolute URL | Production domain | Matches sitemap |
|-----------|----------------|--------------|-------------------|-----------------|
| Homepage | Yes (`DocumentMeta`) | Yes | `canadawashrooms.ca` | Yes |
| Editorial static | Yes | Yes | Apex | Yes (when in sitemap) |
| Province | Yes | Yes | Apex | Yes |
| City | Yes | Yes | Apex | **No** (not in sitemap) |
| Category hub | Yes | Yes | Apex | Yes |
| Landing (segment×city) | Yes | Yes | Apex | Partial |
| Provider | Yes | Yes | Apex | **No** (not in sitemap) |

**Flags:**

1. **www vs apex:** App canonicals use `https://canadawashrooms.ca`. Sitemap on production uses apex (correct). Local generator script still references `www` (stale).
2. **Dual URL taxonomy:** Category hubs (`/construction-jobsites/`) and landing pages (`/construction-portable-washrooms/calgary/`) are **different URL families** for the same segment — not duplicate canonicals, but increases template surface area.
3. **Trailing slash:** Canonical builder normalizes trailing slashes. React Router accepts paths with or without trailing slash, but HTTP 404 occurs before client routing without SPA rewrite.
